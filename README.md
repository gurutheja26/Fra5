# Credit Card Fraud Detection — Vercel Deployment

## Why the original deploy 404'd
Vercel does not support R as a runtime. A repo containing only `main.R` and a
README has nothing Vercel can build or serve, so every route 404s. This
template replaces the R script with an equivalent Python pipeline that Vercel
*can* run as a serverless function.

## Project structure
```
fraud-vercel/
├── api/
│   ├── index.py      # Flask serverless function (prediction API)
│   └── model.pkl      # trained model + scaler (generated by model/train_model.py)
├── model/
│   └── train_model.py # run locally to (re)train and produce model.pkl
├── public/
│   └── index.html     # simple test UI
├── requirements.txt
└── vercel.json
```

## This matches your actual main.R pipeline
`model/train_model.py` is a direct Python port of your R script's logistic
regression path:
- drops the `Time` column (`data1 = df[,-c(1)]`)
- scales only `Amount` (`df$Amount = scale(df$Amount)`)
- 80/20 stratified train/test split (`sample.split(..., SplitRatio=0.80)`)
- logistic regression (`glm(Class ~ ., family="binomial")`)

**Important:** `api/model.pkl` currently ships trained on a small **synthetic
demo dataset** (same 30 columns, fake values) just so the API works out of
the box. To train on your real data:

1. Download `creditcard.csv` (Kaggle "Credit Card Fraud Detection" dataset)
   and place it in the `model/` folder.
2. ```bash
   cd model
   pip install pandas scikit-learn joblib
   python train_model.py
   ```
3. This overwrites `api/model.pkl` with a model trained on your real data,
   and prints accuracy/AUC just like your R script did.
4. Redeploy (`git add`, `commit`, `push` — Vercel auto-redeploys).

## Calling the API
`api/index.py` expects a JSON body with keys `V1` through `V28` and `Amount`
(the same columns as `creditcard.csv`, minus `Time` and `Class`). The test
page at `public/index.html` has a pre-filled JSON example you can edit.

## Deploying to Vercel
1. Push this folder structure to your GitHub repo (root of repo, not nested).
2. In Vercel: **Add New Project → Import** your repo.
3. Framework preset: **Other** (do not let it auto-detect).
4. Leave build command / output directory blank — `vercel.json` handles routing.
5. Deploy.

## Testing after deploy
- Visit `https://<your-project>.vercel.app/` → test UI form.
- Or call the API directly:
  ```bash
  curl -X POST https://<your-project>.vercel.app/api/predict \
    -H "Content-Type: application/json" \
    -d '{"amount": 450, "hour_of_day": 2, "distance_from_home": 40, "num_transactions_last_hour": 8}'
  ```

## If you'd rather keep it in R
Deploy instead to a platform with native R support:
- **shinyapps.io** — easiest for Shiny apps, free tier available
- **Posit Connect**
- **Render / Railway / Fly.io** — via a Dockerfile with R installed

Vercel is not an option for pure R workloads.
